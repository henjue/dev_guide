服务端规范

1、预发布或正式发布分支代码有更新必须提供相对于上一次版本相关变动说明，包括但不限于依赖变更，目录变化，数据库升级，配置文件变化等等

2、预发布或正式发布分支代码有更新必须通过至少一种公司内部认可的联系方式(如：邮件、电话、微信等)通知运维人员有更新
